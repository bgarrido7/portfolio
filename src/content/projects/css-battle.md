CSSBattle is a daily(ish) coding challenge where you try to recreate visual targets using the smallest and most efficient amount of CSS possible. It’s all about precision, creativity, and sometimes completely abusing CSS in ways it was probably never intended for.

I’m a frontend developer, so naturally CSS is something I work with all the time—and honestly, I kind of love it.

Even if it has *that reputation*.

You know the one: you move one thing slightly to the left, and suddenly everything breaks in a completely unrelated way. Layouts shift, spacing disappears, and somehow a `div` from three components ago is now ruining your life.

![CSS moment](https://i.gifer.com/1oUQ.gif)

It’s frustrating for most people… but weirdly fun for me.

CSSBattle takes that feeling and turns it into a game. Instead of fighting CSS, you’re *playing* with it—trying to understand it better, push it further, and sometimes just accept that yes, absolute positioning might fix everything (or destroy it completely).

It’s one of those things I genuinely enjoy coming back to, because it constantly reminds me how powerful (and chaotic) CSS can be.
